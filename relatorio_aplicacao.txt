# Relatório Técnico: Sistema RAG para Valoração de Tecnologias CTA

## 1. Visão Geral da Aplicação
Esta é uma aplicação backend que implementa um sistema RAG (Retrieval Augmented Generation) especializado em valoração de tecnologias relacionadas ao Patrimônio Genético Nacional e Conhecimentos Tradicionais Associados. O sistema utiliza técnicas avançadas de processamento de linguagem natural e recuperação de informações para fornecer respostas precisas e contextualizadas.

## 2. Arquitetura do Sistema

### 2.1 Componentes Principais
- **API REST (app.py)**: Interface principal usando FastAPI
- **Sistema RAG (rag.py)**: Núcleo do processamento de documentos e geração de respostas
- **Banco de Dados**: PostgreSQL para armazenamento de documentos e vetores
- **LLM**: Integração com modelo Llama 3 70B via API NVIDIA
- **Embeddings**: Modelo Multilingual E5 Large Instruct

### 2.2 Fluxo de Dados
1. Upload de documentos
2. Processamento e chunking de texto
3. Geração de embeddings
4. Armazenamento em banco vetorial
5. Recuperação contextual
6. Geração de respostas

## 3. Funcionalidades Principais

### 3.1 Processamento de Documentos
- Suporte atual para arquivos PDF
- Extração de texto com PyPDF2
- Chunking usando NLTK
- Tamanho de chunk: 800 caracteres com 100 de sobreposição
- Limpeza e normalização de texto

### 3.2 Sistema de Embeddings
- Modelo: intfloat/multilingual-e5-large-instruct
- Configurações:
  * Device: CPU
  * Normalização de embeddings ativada
  * Otimizado para português

### 3.3 Banco de Dados
- PostgreSQL com extensão vetorial
- Tabelas principais:
  * documentos_originais: Armazena documentos fonte
  * chunks_vetorizados: Armazena texto segmentado e embeddings

### 3.4 API Endpoints
1. **/upload**
   - Recebe documentos PDF
   - Processa e vetoriza conteúdo
   - Armazena no banco de dados

2. **/chat**
   - Recebe queries em linguagem natural
   - Recupera contextos relevantes
   - Gera respostas baseadas em documentos

## 4. Tecnologias Utilizadas

### 4.1 Principais Bibliotecas
- FastAPI: Framework web
- PyPDF2: Processamento de PDFs
- NLTK: Processamento de linguagem natural
- LangChain: Ferramentas RAG
- HuggingFace: Modelos de embeddings
- Psycopg2: Conexão PostgreSQL

### 4.2 Modelos e APIs
- LLM: meta/llama3-70b-instruct
- Embeddings: intfloat/multilingual-e5-large-instruct
- API NVIDIA para inferência

## 5. Configurações do Sistema

### 5.1 Parâmetros LLM
- Temperature: 0.2
- Max tokens: 1024
- Top_p: 0.9
- Frequency penalty: 0.3
- Presence penalty: 0.2

### 5.2 Configurações de Chunking
- Tamanho do chunk: 800 caracteres
- Sobreposição: 100 caracteres
- Método: NLTK sentence tokenizer

## 6. Casos de Uso

### 6.1 Principais Aplicações
- Análise de documentos de valoração
- Consultas sobre metodologias de valoração
- Extração de informações técnicas
- Suporte à decisão em valoração

### 6.2 Tipos de Consultas Suportadas
- Metodologias de valoração
- Aspectos regulatórios
- Análise de tecnologias
- Cálculos e métricas

## 7. Limitações Atuais
- Processamento apenas de PDFs
- Sem suporte a tabelas estruturadas
- Chunking fixo
- Sem cache de embeddings
- Sem validação de respostas

## 8. Requisitos do Sistema
- Python 3.8+
- PostgreSQL
- Acesso à API NVIDIA
- Dependências listadas em requirements.txt
- Mínimo 8GB RAM recomendado

## 9. Variáveis de Ambiente Necessárias
- API_KEY_NVIDIA
- DATABASE_URL
- PORT (default: 8001)
